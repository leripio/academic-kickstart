Como estimar modelos hierarquicos de maneira eficiente

Última atualização em Jul 29, 2020 Data Science, Previsão

Modelos hierárquicos são aqueles que utilizam alguma estrutura de desagregação para obter previsões. A abordagem mais comum, conhecida como bottom-up, consiste em ajustar um modelo específico para cada quebra da série principal a fim de obter a projeção do total.

A grande vantagem é poder explorar um conjunto maior de informação e capturar dinâmicas individuais que se perderiam ao estimar diretamente o agregado. Entretanto, é preciso cuidado para não desagregar demais e adicionar apenas ruído às projeções. A minha recomendação é modelar individualmente apenas aquelas séries para as quais existem boas variáveis explicativas ou, caso seja um modelo univariado, séries cujos componentes – tendência-ciclo e sazonalidade – sejam fáceis de extrair.

No que diz respeito à implementação, um desafio se coloca: é preciso estimar um modelo para cada grupo de dados. Existe uma forma bem conveniente de lidar com esse tipo de tarefa que utiliza as funções nest do pacote tidyr e map do pacote purrr.

No exemplo que segue, vamos assumir que o objetivo seja projetar a Produção Industrial (PIM-PF) brasileira a partir das estimativas para cada UF. Isto é, precisamos ajustar 14 modelos (nem todas as UF’s estão contempladas na pesquisa), obter as projeções de cada um e, em seguida, computar o agregado.

Passo 1: carregar pacotes e importar os dados

library(sidrar)
library(tidyverse)
library(fable)
library(tsibble)

pim_ibge <- sidrar::get_sidra(api = "/t/3653/n3/all/v/3135/p/all/c544/129314/d/v3135%201")

pim_ibge <- pim_ibge %>%
  
  dplyr::select(date = `Mês (Código)`,
                uf   = `Unidade da Federação`,
                valor = Valor) %>%
  
  dplyr::mutate(date = lubridate::ymd(paste0(date, "01")))

Passo 2: colocar os dados no formato nest

pim_ibge_nest <- pim_ibge %>%
  
  dplyr::group_by(uf) %>%
  
  tidyr::nest()

Repare no frame criado pela função nest. Diferente das estruturas convencionais, ele permite armazenar objetos dentro das células. Na prática, isso funciona como uma lista sobre a qual podemos usar o map para generalizar operações (fazer um loop).

Vamos usar esse recurso para ajustar um modelo ARIMA automático para cada uma das UF’s. É possível criar modelos diferentes para cada uma das UF’s. Para isso, precisaríamos incluir uma lista com as especificações na estrutura do map.

Passo 3: estimar os modelos

pim_ibge_model <- pim_ibge_nest %>% 
  
  dplyr::mutate(modelos = purrr::map(.x = data, 
                                     .f = function(x){
    
    x %>%
      
      dplyr::mutate(date = tsibble::yearmonth(date)) %>%
      
      as_tsibble() %>%
      
      model(modelo = ARIMA(valor)) %>%
      
      forecast(h = 12)
    
  }))

As projeções trazem informações sobre a distribuição das projeções para cada UF, o que permite plotar gráficos contendo intervalos como este abaixo.

Sobre esse aspecto, cabe notar que para termos intervalos de previsão para o agregado devemos estimar a matriz de variância-covariância dos erros das projeções individuais. Não vamos abordar isso neste post e, portanto, vamos reter apenas o valor central (média).

pim_ibge_prev <- pim_ibge_model %>%
  
  dplyr::mutate(previsao = purrr::map(.x = modelos, .f = function(x){
    
    x %>% 
      
      as_tibble() %>%
      
      dplyr::select(date, previsao = .mean)
    
  }))

Passo 4: computar o agregado

Para finalizar, vamos desaninhar os nossos dados com unnest para voltar ao formato convencional e colocar o frame no formato wide. Com as projeções para cada UF em uma coluna, podemos computar a projeção para o Brasil fazendo a média das linhas. Note que isso pressupõe que cada UF tem o mesmo peso, o que é uma simplificação.

A ressalva aqui é que utilizei a função c_across que é exclusiva da versão 1.0 do dplyr. Em tempo, essa nova versão é bem mais robusta para trabalhar com operações rowwise.

pim_ibge_final <- pim_ibge_prev %>%
  
  dplyr::select(uf, previsao) %>%
  
  tidyr::unnest(cols = c(previsao)) %>%
  
  tidyr::pivot_wider(names_from = uf, values_from = previsao) %>%
  
  dplyr::rowwise() %>%
  
  dplyr::mutate(Brasil = mean(c_across(cols = -date)))
